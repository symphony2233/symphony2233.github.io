架构说明：

微服务依赖于基础工程，基础工程继承父工程
父工程对依赖包的版本进行管理、对子工程进行聚合管理
基础工程：提供基础类库和工具类库
maven中的继承：一个模块可以从另一个模块中基础配置和依赖
依赖：一个模块可以使用另一个模块的功能或代码库

module A依赖moduleB，而moduleB继承与module C —》 module A 间接继承 module C 的依赖和配置

一个微服务分成了api、service、model

api：为前端提供接口
service：为接口工程提供业务支撑
model：数据模型工程，存储数据模型类、数据传输类型等

接口

查询一般用post

dto:数据传输对象：用于在不同层之间传输数据
vo：表示值或数值的对象，通常不可变 ，用于展示给用户，不包含业务逻辑方法
po：持久化对象，映射数据库表实体
query：查询对象

分页插件的原理：
首先分页参数放到ThreadLocal中，拦截执行的sql，根据数据库类型添加对应的分页语句重写sql，例如：(select * from table where a) 转换为 (select count(*) from table where a)和(select * from table where a limit ,)
计算出了total总条数、pageNum当前第几页、pageSize每页大小和当前页的数据，是否为首页，是否为尾页，总页数等。
